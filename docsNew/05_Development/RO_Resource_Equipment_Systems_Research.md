## 2. Equipment Breaking System

### Two Distinct Breaking Contexts

#### A. Refine Breaking (Upgrading)
- Attempting to refine past safe levels can **permanently destroy** the item
- Safe levels: Weapon Lv1 = +7, Lv2 = +6, Lv3 = +5, Lv4 = +4
- **Destroyed items are gone forever** - cards, refine level, everything lost
- Cannot be repaired - this is permanent destruction

#### B. Combat Breaking (Battle Damage)
- Equipment becomes **damaged/broken** but is NOT destroyed
- Broken items show **red text** in their name
- Broken items **cannot be equipped** (they are unequipped when broken)
- All bonuses from the broken item are lost (ATK, DEF, card effects, etc.)
- Broken items CAN be stored in cart/Kafra
- Broken items CAN be repaired

### Combat Break Triggers

#### Natural Break Rate (Normal Attacks)
- Controlled by `battle_config.equip_natural_break_rate` (default: **0** on official servers)
- This means on official servers, normal attacks do NOT break equipment
- Modified by `equip_self_break_rate` multiplier (default: 100 = 100%)

#### Self-Break (Your Own Equipment)
- **Over Thrust (SC_OVERTHRUST):** Adds 0.1% chance of breaking YOUR equipped weapon per attack
- **Max Over Thrust (SC_MAXOVERTHRUST):** Also adds weapon self-break chance
- Naturally unbreakable weapon types are EXEMPT: Axes, Maces, Books, Staves
- Rate formula: `base = equip_natural_break_rate; if(SC_OVERTHRUST) base += 10; if(SC_MAXOVERTHRUST) base += 10;`
- Final rate: `base * equip_self_break_rate / 100`

#### Target Break - Card Bonuses (bBreakWeaponRate / bBreakArmorRate)
- `bonus bBreakWeaponRate,n;` - n/100% chance to break enemy's weapon per attack
- `bonus bBreakArmorRate,n;` - n/100% chance to break enemy's armor per attack
- These stack with other break chances
- **MasterSmith Card (ID 4361):** `bBreakWeaponRate,1000; bBreakArmorRate,700;` = **10% weapon break, 7% armor break** per hit
- Controlled by `equip_skill_break_rate` multiplier (default: 100 = 100%)

#### Target Break - Skills
| Skill | Weapon Break | Armor Break | Notes |
|-------|-------------|-------------|-------|
| Shattering Strike / Meltdown (WS_MELTDOWN) | Lv*1% (1-10%) | Lv*0.7% (0.7-7%) | Duration: 10+Lv*5 seconds. On monsters: reduces ATK/DEF instead. Does not affect Boss Protocol. |
| Acid Terror (AM_ACIDTERROR) | - | (Lv-1)*10+5% (5-45%) | Armor break persists even if damage blocked by Pneuma. Kyrie/Guard block both damage and break. |
| Acid Demonstration (CR_ACIDDEMONSTRATION) | Lv% | Lv% | PvP only break effect |

**Break chance formula for skill-based breaks (e.g., Shatter Storm):**
```
rate = max(skill_lv * 500, (caster_DEX * skill_lv * 10) - (target_AGI * 20))
```

#### Equipment Types That Cannot Break (Naturally Unbreakable)
- Bare fists (no weapon equipped)
- Axes
- Maces (most)
- Books
- Staves/Rods

### Protection Against Breaking

| Method | Effect |
|--------|--------|
| `bUnbreakableWeapon` | Weapon cannot be broken by ANY means |
| `bUnbreakableArmor` | Armor cannot be broken by ANY means |
| `bUnbreakable,n` | Reduces break chance of ALL equipment by n% |
| Chemical Protection (Weapon/Armor/Shield/Helm) | SC_CP_WEAPON, SC_CP_ARMOR etc. - prevents strip AND break |
| Golem Card (weapon) | `bUnbreakableWeapon` |
| Restless Dead Card | `bUnbreakableWeapon` + `bUnbreakableArmor` |

**Note:** `bUnbreakable` items can still be broken during refining - only combat breaking is prevented.

### Repairing Broken Equipment

#### Method 1: Blacksmith Skill - Weapon Repair
- SP Cost: 30, Cast time: 5s total
- Range: 2 cells
- Materials by weapon level:
  - Lv1 weapons: Iron Ore
  - Lv2 weapons: Iron
  - Lv3 weapons: Steel
  - Lv4 weapons: Rough Oridecon
  - Armor: Steel

#### Method 2: Repairman NPC
- Cost: **5,000 Zeny** per broken item
- Detects all broken items in inventory
- Located in most major cities:
  - Prontera (Upgrading Hall)
  - Alberta (Merchant Guild)
  - Morroc (Pyramids)
  - Payon (Upgrading Area)
  - Geffen (Armory)
  - Al de Baran (Armory)
  - Juno (Upgrading Hall)
  - Lighthalzen (Armory)

### Status Effects
- `SC_BROKENWEAPON` (ID 50) - Visual status icon, equipment treated as if not worn
- `SC_BROKENARMOR` (ID 51) - Visual status icon, equipment treated as if not worn
- Both configured with `SCB_NONE` (no direct stat modification - the stat loss comes from the item being treated as unequipped)

### Battle Config Settings
```
equip_natural_break_rate: 0     // Base break rate per attack (0 = disabled on official)
equip_self_break_rate: 100      // Multiplier for self-break (Over Thrust etc.)
equip_skill_break_rate: 100     // Multiplier for target break (Acid Terror, Meltdown, cards)
break_mob_equip: no             // Can players break monster equipment? (official: no)
```

## 3. Movement Speed System

### Base Speed
- **Default player walk speed: 150ms per cell** (0.15 seconds)
- This means lower ms = faster movement
- `DEFAULT_WALK_SPEED = 150` (in rAthena mmo.hpp)
- `MIN_WALK_SPEED = 20` (speed cap - fastest possible)
- `MAX_WALK_SPEED = 1000` (slowest possible)

### Speed Calculation Formula

In rAthena, speed is stored as **ms per cell** (lower = faster). The formula:
```
// bSpeedRate: only highest value used, does NOT stack with SC_SPEEDUP0/SC_SPEEDUP1
final_speed = base_speed * (100 - speed_rate) / 100

// Then cap between MIN and MAX
final_speed = cap_value(final_speed, MIN_WALK_SPEED, MAX_WALK_SPEED)
```

**Important:** `bSpeedRate` is a **percentage reduction** of the ms-per-cell value.
- `bSpeedRate,25` means speed becomes `150 * 75/100 = 112ms/cell` (not exactly 0.11, but close)
- Only the HIGHEST bSpeedRate is used (they do NOT stack with each other)
- Does NOT stack with SC_SPEEDUP0 (Speed Potion) or SC_SPEEDUP1

### Speed Modifiers (Pre-Renewal Classic)

| Source | ms/cell | ~% of Normal | Notes |
|--------|---------|--------------|-------|
| Base (no modifiers) | 150ms | 100% | |
| Increase AGI (SC_INCREASEAGI) | ~110ms | ~136% | Buff skill, stacks with Peco |
| Peco Peco Ride | ~110ms | ~136% | Knight/Crusader mount |
| Increase AGI + Peco Peco | ~70ms | ~214% | These two DO stack |
| Speed Potion (SC_SPEEDUP1) | ~60ms | ~250% | 5 second duration, cancels speed reductions, does NOT stack with anything else |
| Moonlight Flower Card | bSpeedRate,25 | ~133% | Footgear card, does not stack with other bSpeedRate |

### Items That Provide bSpeedRate (Pre-Renewal)
- **Moonlight Flower Card (4131):** `bSpeedRate,25` (footgear)
- Various equipment sets provide equivalent bonuses

### Speed Reductions
| Source | Effect | Notes |
|--------|--------|-------|
| Pushcart | Speed reduced by (50 - 5*SkillLv)% | At Lv10: only 0% penalty |
| Curse (SC_CURSE) | Speed +300ms (much slower) | Also: ATK-25%, LUK=0 |
| Decrease AGI (SC_DECREASEAGI) | Reduces walkspeed | Buff/debuff |
| Quagmire | Cancels most speed/ASPD buffs | Ground effect |
| Slow Grace / Battle Chant | Various reductions | |

### Stacking Rules Summary
1. **bSpeedRate** - Only highest value applies. Does NOT stack with Speed Potion effects.
2. **bSpeedAddRate** - DOES stack with other bonuses (additive).
3. **Increase AGI + Peco Peco** - These two stack with each other.
4. **Speed Potion** - Overrides everything, does NOT stack with ANY other speed bonus. Cancels speed reductions. 5 second duration.
5. **Speed cap:** Minimum 20ms/cell (~750% speed), Maximum 1000ms/cell (~15% speed).

## 4. Intravision / Hidden Detection (Maya Purple Card)

### Maya Purple Card (ID 4198)
**Script:** `bonus bIntravision;`
**Slot:** Headgear
**Effect:** "Enable its user to detect hidden enemies"

### What bIntravision Does
- **Passive** - always active while the card is equipped
- Allows the wearer to **see** Hiding and Cloaking players/monsters
- Detection is **client-side visual** - the hidden player appears translucent to the Intravision user
- **No range limit** - works within the player's screen view
- Does NOT reveal the hidden player to other party members

### Detection Capabilities
| Stealth Type | Detected by Intravision? | Notes |
|-------------|-------------------------|-------|
| Hiding (TF_HIDING) | **YES** | Fully visible to wearer |
| Cloaking (AS_CLOAKING) | **YES** | Fully visible to wearer |
| Chase Walk (ST_CHASEWALK) | **NO** | Chase Walk is "undetectable by all detecting skills" |

### Chase Walk Special Rules
Chase Walk (Stalker skill) is specifically designed to be immune to ALL detection:
- NOT revealed by Sight, Ruwach, Detect, Improve Concentration
- NOT revealed by Maya Purple Card / Intravision
- NOT revealed by Insect/Demon/Boss race detection
- The ONLY ways to hit someone in Chase Walk are AoE skills that happen to target their cell

### Other Detection Methods (for Hiding/Cloaking)
| Method | Reveals Hiding? | Reveals Cloaking? | Duration |
|--------|----------------|-------------------|----------|
| Sight (MG_SIGHT) | Yes | Yes | Continuous AoE around caster |
| Ruwach (AL_RUWACH) | Yes | Yes | Continuous AoE, deals damage |
| Detect (RG_DETECTION) | Yes | Yes | Instant reveal |
| Improve Concentration | Yes | Yes | Instant reveal on cast |
| Arrow Shower / Heaven's Drive / Earth Spike | Yes (if hit) | Yes (if hit) | AoE damage reveals |
| Box of Sunlight (item) | Yes | Yes | One-time use |
| Insect/Demon/Boss monsters | Yes | Yes | Passive monster detection |

### Interaction with Sight
- Sight and Intravision serve different purposes
- **Sight** actively reveals to everyone in range (AoE effect)
- **Intravision** only reveals to the wearer (personal visual)
- They can be used together - Sight reveals to the whole party, Intravision ensures personal vision

### How bAddItemGroupHealRate Works

**Syntax:** `bonus2 bAddItemGroupHealRate,ig,n;`
**Effect:** Increases HP recovered by `n%` for ALL items belonging to item group `ig`.

Example: A card with `bonus2 bAddItemGroupHealRate,IG_Potion,50;` would make Red/Orange/Yellow/White Potions heal 50% more HP.

The bonus is **additive** with other healing modifiers (VIT recovery bonus, etc.).

## 6. Monster Drop Item Groups (bAddMonsterDropItemGroup)

### How It Works

**Syntax:** `bonus2 bAddMonsterDropItemGroup,ig,n;` - n/100% chance when killing ANY monster
**Syntax:** `bonus3 bAddMonsterDropItemGroup,ig,r,n;` - n/100% chance when killing monster of race `r`

### Mechanics

1. When a monster dies, the game rolls for each `bAddMonsterDropItemGroup` bonus
2. If the roll succeeds (n/100% chance), ONE random item from the group is selected
3. The **actual drop rate** of each item within the group is defined in `item_group_db.yml` (the rates shown in the group tables above)
4. **Official behavior:** `n` is always `10000` (100%) for all official items - the group always activates, but the internal group rates determine which specific item drops

### Negative Values
If `n` is negative: `chance = -n * (killed_mob_level / 10) + 1` - scales with monster level.

### Limits
- Maximum of **10** `bAddMonsterDropItemGroup` bonuses can be active simultaneously
- Extras beyond 10 are silently ignored

### Official Card/Item Examples
- **Gaia Sword (weapon):** `bonus2 bAddMonsterDropItemGroup,IG_Jewel,10000;` + `bonus2 bAddMonsterDropItemGroup,IG_Ore,10000;`
  - 100% activation rate per kill, then internal group rates determine the specific jewel/ore

### Key Item Groups Used by Drop Bonuses
| Group | Contains |
|-------|----------|
| IG_Jewel | Gemstones, crystals |
| IG_Ore | Ores, metals |
| IG_Recovery | Healing herbs, potions, food |
| IG_Potion | Red/Orange/Yellow/White Potions |
| IG_Food | Various food items |

## Implementation Notes for SabriMMO

### Priority for Implementation
1. **Gemstone consumption** - High priority (many skills already in game need catalysts)
2. **Movement speed system** - High priority (Increase AGI already exists as buff)
3. **Intravision** - Low priority (requires Hiding/Cloaking implementation first)
4. **Equipment breaking** - Medium priority (adds depth to combat but complex)
5. **Item groups** - Medium priority (useful for future card effects)
6. **Monster drop groups** - Low priority (future card bonus support)
