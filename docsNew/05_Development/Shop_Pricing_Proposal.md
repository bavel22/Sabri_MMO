## Pricing Formula (proposed)

### Stat tier table (per item, total of STR+AGI+VIT+INT+DEX+LUK)

| Total stats | Minimum floor |
|---|---|
| 0 | 0 (no bonus) |
| 1 | 200,000z |
| 2 | 600,000z |
| 3 | 1,500,000z |
| 4 | 4,000,000z |
| 5+ | 4M × 1.5^(n-4), capped at 2B |

### Weapon formula

```
wLvFactor = {1:30, 2:150, 3:1000, 4:4000}    // 5x more aggressive than v1
atkBase   = ATK * wLvFactor[weapon_level]
stat      = statTierTable(total_stats)
hpsp      = floor(maxHP/100)*50k + floor(maxSP/10)*50k
element   = (holy|dark|ghost|undead): 75k
            (fire|water|wind|earth|poison): 25k
final     = max(existing_buy_price, (atkBase + stat + hpsp + element) * 2^slots)
```

### Armor formula

```
defBase   = DEF * 1000
stat      = statTierTable(total_stats)
hpsp      = same as weapon
element   = same as weapon
levelMult = max(1, equip_level_min / 20)
final     = max(existing_buy_price, (defBase + stat + hpsp + element) * levelMult * 2^slots)
```

### Consumable formula (HP/SP heal items only)

```
hp_value = HP * (1 + HP/100)
sp_value = SP * 100
cure     = num_cured_statuses * 100
final    = max(existing_buy_price, hp_value + sp_value + cure, 100)
```

### Floor approach

Every item gets `final = max(existing_buy_price, computed_floor)`.
**Nothing ever gets cheaper.** Items currently priced at canonical rAthena values
higher than the floor stay at their canonical price.
