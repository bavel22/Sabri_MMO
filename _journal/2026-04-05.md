### Session 1: Options Menu Expansion
**Goal**: Player-configurable settings menu with persistence.
**Accomplished**: Full options menu — 14 settings across 4 sections, SaveGame persistence, FPS overlay, wired into ESC menu.

- New `OptionsSubsystem` + `SOptionsWidget` (4 sections: Display / Interface / Camera / Gameplay)
- `UOptionsSaveGame` UPROPERTY backing — slot `"SabriMMO_Options"` → `{Saved}/SaveGames/SabriMMO_Options.sav`
- 9 target subsystems wired to read settings: `DamageNumberSubsystem` (`bDamageNumbersEnabled`, `bShowMissText`), `WorldHealthBarSubsystem` (`bShowEnemyBars`), `NameTagSubsystem` (`bShowPlayerNames`, `bShowEnemyNames`), `CameraSubsystem` (`ZoomSpeed`, `RotationSensitivity`), `PostProcessSubsystem` (`SetBrightness`), and gameplay flags read by `PlayerInputSubsystem`/`TradeSubsystem`/`PartySubsystem`

- Opens from ESC menu

**Key learning**: GConfig with custom ini files does NOT reliably load in standalone — must use SaveGame for any custom persistence. Saved as feedback memory `feedback-savegame-over-gconfig.md`.

### Session 2: Character Positioning Fix
**Goal**: Stop the character floating above terrain / clipping through objects when moving north (carried over from 04-03 and 04-04).
**Accomplished**: Root cause identified and fixed across local + remote players + sprite VFX spawn points.

- `SabriMMOCharacter` now caches a `TWeakObjectPtr<ASpriteCharacterActor> LocalSprite` set in `BeginPlay()`
- Name tag re-registered against the sprite actor instead of the pawn (so it tracks the smooth ground-snapped Z, not the gravity-driven capsule Z)
- `OtherCharacterMovementComponent::TickComponent` now uses a fixed `HalfHeight = 96` (matching `SabriMMOCharacter::InitCapsuleSize(42, 96)`) so the sprite Z subtraction produces ground level consistently across all remote players
- `SpriteCharacterActor` ground-snap compensates for `SpriteDepthOffset` (125 units): `Snapped.Z -= (-CamFwd.Z) * SpriteDepthOffset`
- All VFX/particles on sprite enemies now spawn at `ActorZ + SpriteSize.Y * 0.5` (visual center), NOT raw actor origin (which sits underground)
- Ground snap consolidated to a shared constant `GSpriteCameraDepthOffset` so projection and snap stay in lockstep

**Key learning**: Billboard `SpriteDepthOffset` of 125 units causes vertical lift; ground snap MUST compensate or characters/effects appear floating or buried. Saved as `feedback-billboard-ground-snap.md`.

### Session 3: Damage Number Overhaul + Hit Impact System
**Goal**: Make auto-attack combat feel exciting — rewrite damage numbers to RO Classic spec and add hit impact polish (flash, particles, starburst, combo total, flinch, sounds).
**Accomplished**: Massive multi-phase session — initial RO rewrite + Status color fix + all 6 hit impact phases.

**Phase A — RO Classic damage number rewrite** (`SDamageNumberOverlay`, `DamageNumberSubsystem`):
- Replaced ease-out vertical rise with **parabolic sine arc + horizontal drift + scale shrink**
- Per-type animation curves: damage (arc + drift + 3.0x→0 shrink, 1.5s), miss/dodge/block (straight rise, fixed 0.7x, 0.8s), heal (stationary then rise, 2.5x→0.8x hold, 1.5s)
- RO Classic colors: normal=white, crit=yellow, player-hit=pure red, heal=pure green (swapped from previous inverted assignment)
- Element tinting kept for normal combat damage (white + 40% blend)
- Stack/spread logic relaxed since drift naturally separates hits
- Per-entry `Lifetime` and `DriftDirection` (alternating ±1 for visual variety)

**Phase B — Status effect damage coloring fix**:
- Added optional `CustomColor` parameter through `SpawnDamagePop()` → `AddDamagePop()`
- `HandleStatusTick()` now passes the full status color directly (poison=purple, bleeding=red, stone=brown) instead of weak 40% element blend
- Status applied text ("Poisoned!", "Stunned!") confirmed already using miss-animation path

**Phase C — Hit impact features (all 6 phases)**:
1. **Hit Flash** — `SpriteCharacterActor::PlayHitFlash()`: saves layer tints, sets all to `(3,3,3,1)` overbright for 150ms, restores in `Tick`. Triggered from `CombatActionSubsystem::HandleCombatDamage` after target resolve. Skipped on miss/dodge.
2. **Hit Particles** — `SkillVFXSubsystem::SpawnAutoAttackHitEffect()` loads `NS_AutoAttackHit` (located at `/Game/Variant_Combat/VFX/`). Spawns at visual center (`ActorZ + SpriteSize.Y * 0.5`), white normal / yellow crit, scale 0.8x normal / 1.5x crit. Only fires on `combat:damage`, not `skill:effect_damage`.
3. **Crit Starburst** — Texture loaded into `DamageNumberSubsystem`, passed to `SDamageNumberOverlay` as `FSlateBrush` (NOT deprecated `FSlateDynamicImageBrush`). Rendered behind digit group in `OnPaint` for `CriticalDamage`/`PlayerCritHit` types. Starburst PNG generated via `_scripts/generate_starburst.py` + `generate_starburst_red.py`.
4. **Combo Total** — `FComboTracker` map keyed `"{attackerId}_{targetId}_{skillId}"`. Multi-hit skills (`totalHits > 1`) buffer damage; on `hitNumber >= totalHits` spawn `EDamagePopType::ComboTotal` (3.0s lifetime, big yellow, rapid pop-in 0→3.5x in 50ms, slow rise, no drift, slower fade).
5. **Target Flinch** — `SpriteCharacterActor::PlayHitAnimation()` triggers `Hit` animation state. `enemy:move` events now guarded against `Hit`/`Attack`/`Death` states so the flinch isn't immediately overridden. Skipped on miss/dodge/perfectDodge.
6. **Hit Sounds** — First audio in the project. 5 WAVs generated via `_scripts/generate_hit_sounds.py`, loaded into `CombatActionSubsystem` as `TArray<USoundBase*> NormalHitSounds` + `USoundBase* CritHitSound`. Random variant + ±5% pitch jitter via `UGameplayStatics::PlaySoundAtLocation` at target position.

## Key Decisions

- **SaveGame over GConfig** — Standalone mode does not reliably load custom ini files. All player settings persistence uses `USaveGame` subclasses going forward. (`feedback-savegame-over-gconfig.md`)
- **Fixed HalfHeight=96 in remote movement component** — Matching the local capsule size constant produces consistent ground-snap behavior across all remote players, regardless of how the engine reports their capsule.
- **Name tag attached to sprite, not pawn** — Pawn Z is gravity-driven, sprite Z is smooth-snapped. UI must track the smooth Z to avoid jitter.
- **Status damage uses CustomColor, not element blend** — 40% HSV blend was too subtle. Full status colors are visually distinct against busy backgrounds.
- **Crit starburst as `FSlateBrush`, not `FSlateDynamicImageBrush`** — Latter is deprecated in UE5.7 and triggers ensure failures. Source `UTexture2D` kept alive via UPROPERTY on subsystem. (Reinforces `feedback-fslatebush-not-dynamic.md`.)
- **Hit particles spawn at visual center, not actor origin** — Sprite enemy actor origin is below ground due to billboard depth offset; particles must use `ActorZ + SpriteSize.Y * 0.5`.
- **enemy:move guarded against Hit/Attack/Death** — Otherwise the next position update immediately overrides the flinch animation, defeating Phase 5.
