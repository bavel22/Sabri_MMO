## What Got Done

### Enemy Enumeration Scripts
- **`_prompts/build_enemy_list.js`** (142 lines) — one-off generator:
  - Requires `server/src/ro_monster_templates.js`, iterates every monster template
  - Scans `2D animations/3d_models/enemies/*.glb` on disk, matches filenames to templates via a hand-curated `GLB_ALIAS` map (handles typos like `drainiliar` → `drainliar`, `eggrya` → `eggyra`, `orc_warrior` → `ork_warrior`, `boa` → `snake`, `marlin` → `marse`)
  - Scans `2D animations/atlas_configs/*_v2.json` to detect which monsters have actual atlas coverage
  - Pulls a hand-maintained `PRESET_SUGGESTIONS` table (12 animation presets: blob / caterpillar / rabbit / egg / frog / tree / bird / flying_insect / bat / quadruped / plant / biped_insect) keyed by template name
  - Emits structured JSON to `_prompts/.enemy_dump.json` with fields: `{ id, key, name, level, race, element, size, monsterClass, preset, glb, hasAtlas, spriteTint }`
- **`_prompts/build_enemy_markdown.js`** (103 lines) — consumer:
  - Reads `_prompts/.enemy_dump.json`
  - Groups enemies into level buckets: 1-10 / 11-20 / 21-30 / 31-40 / 41-50 / 51-60 / 61-70 / 71-80 / 81-90 / 91-100
  - Emits a GFM table per bucket with columns `Lv | ID | Key | Name | Race | Element | Size | Preset | Sprite`
  - Sprite column resolves to: `done` / `done (tinted)` (if atlas + spriteTint present) / `pending (GLB ready)` (GLB on disk, no atlas yet) / `no GLB`
  - Escapes `|` in names, bolds `MVP`/`BOSS` class tags inline with name

- **Enemy roster regen happens from scripts, not hand-edited** — hand-maintained lists drifted (04-15's resume prompt said "21 monsters with sprites complete" but on-disk atlas count + template spriteClass count had moved).
