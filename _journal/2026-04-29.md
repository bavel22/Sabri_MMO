## What Got Done

### Foliage occlusion threshold in `CreateSpriteMaterial` (02:00)

`client/SabriMMO/Source/SabriMMO/Sprite/SpriteCharacterActor.cpp::CreateSpriteMaterial` — added a `FoliageOcclusionDist` scalar parameter (default `80.f` UU) and rewrote the opacity custom-node:

```hlsl
float Behind = InPixelDepth - InSceneDepth;
float FoliageHide = (Behind > 1.0 && Behind < InFoliageDist) ? 0.0 : 1.0;
return (InAlpha > 0.333) ? FoliageHide : 0.0;
```

- **Behind ≤ 1.0**: nothing in front → sprite draws normally
- **1.0 < Behind < FoliageOcclusionDist (80 UU)**: small occluder (grass, flower, debris) → discard pixel, sprite *vanishes* behind it
- **Behind ≥ FoliageOcclusionDist**: large occluder (wall) → opacity stays 1.0, the existing emissive line-trace path takes over and renders the silhouette through the wall

Why two paths and not just one: the post-process cutout system from 04-06 *needs* `Opacity = 1` when a wall occludes the sprite, otherwise the stencil never gets written and the silhouette can't be drawn. Foliage doesn't need a silhouette — it should fully hide the sprite. The 80 UU threshold separates "ankle-deep grass" from "stone wall."

## Key Decisions

- **Single threshold over per-tag classification** — UE5 doesn't expose a clean per-actor "is this foliage" signal in the material graph. The depth differential is a reliable proxy: most grass clumps are < 50 UU thick, most walls are ≥ 100 UU.
- **Default 80 UU is conservative** — tunable per material instance via `FoliageOcclusionDist` if a zone has unusually thin walls or unusually thick grass.
