### Multi-object detection (17:54 → 18:10)

Some batch 1 assets came back as 2-3 separate meshes — SDXL drew pairs ("two trees side by side") despite the negative prompts, and the Hy3D mesh-gen produced disconnected blobs. Even with `cube_artifact=0.001` they passed the gate.

`_tools/find_multi_object_assets.py` (9,763 bytes) — walks `04b_decimated_untextured.glb` for every asset, runs `trimesh.split()` to count connected components. Flags assets where the second-largest component is > 25% of the largest (anything below that is acceptable noise from auto-decimation).

`_tools/find_multi_object_v2.py` (11,655 bytes) — refined heuristic: also checks centroid spread (multiple components separated by > 30% of the bounding box diagonal) and bounding-box overlap (components that share a tight bbox are likely one logical asset, those that don't are pairs). Output reports:

- `multi_object_report.json` — first-pass list, ~32 assets flagged
- `multi_object_v2.json` — refined list, ~18 assets confirmed multi-object after the spread + overlap checks
- `multi_object_v2.txt` — human-readable summary with per-asset component counts

### Multi-object regen — `hunyuan_v2_regen.py` (18:52)

9,652 bytes. For each flagged asset:
1. Strengthen the negative prompt: prepend "ABSOLUTELY ONE OBJECT, NO PAIRS, NO COPIES, NO DUPLICATES" and append "two objects, three objects, pair, group, cluster of objects, duplicate, copies" with extra weight
2. Re-run multi-input + mesh-gen sweep with 8 seed candidates instead of 4 (more attempts)
3. After mesh-gen, run `trimesh.split()` and keep ONLY the largest component above the 25% threshold
4. Decimate-then-texture as normal

About 14 of the 18 flagged assets recovered; the remaining 4 were genuinely "concept assets" (e.g. "twin lampposts") where the SDXL prompt itself implied a pair.

## Key Decisions

- **Multi-object filtering is post-process, not pre-process** — trying to make SDXL produce single objects 100% of the time is fighting the model. Easier to detect afterward and either re-roll with stricter prompts or keep only the dominant blob.
- **`largest-component-only` is the lever for stubborn pairs** — when 8 reroll attempts can't produce a true single-blob output, just delete the smaller components from the resulting mesh. The textured remainder is usually still usable.
- **Variants follow the base** — if a base asset is single-object after v2 regen, all its color variants automatically inherit (they re-texture the same `04b_decimated_untextured.glb`). Don't need to re-run multi-object detection on the variants.
- **Flat layout for UE5 import** — `Imported_to_UE5/Final/<category>/<asset>.glb` is what the UE5 Python scripts walk. Don't make UE5 navigate `<asset_id>/06_final.glb` — too many directory levels for the editor's bulk-import path.
- **Filed the canonical record + prompt** so future Claude sessions don't have to re-discover this. The 04-27 sprite-quality work proved the value of writing a self-contained re-entry doc.
