## Key Decisions
 Step 1: Mixamo Animations (biggest impact)

  Right now all 4 idle frames are identical because the model is static. Animations make or break the sprites.

  1. Go to mixamo.com → upload Female Mage.glb
  2. Mixamo auto-rigs it (~30s)
  3. Download 4 animations as FBX ("With Skin"):
    - Search "Idle" → pick a breathing/subtle idle
    - Search "Walking" → pick a walk cycle
    - Search "Casting Spell" or "Standing 1H Magic Attack" → for attack
    - Search "Dying" or "Death" → pick a death fall
  4. Save all 4 FBX files to 2D animations/3d_models/animations/mage_f/
  5. I re-render with --anim-dir and you get real animated sprites (idle 4f, walk 8f, attack 6f, death 3f)

  Step 2: Validate with a second class

  Try a hatless class (Swordsman or Archer) to confirm the pipeline handles different body types. Upload another hero ref to
  Tripo3D → GLB → Mixamo → render.

  Step 3: Post-process to final size

  Run the 1024 renders through the post-processor to get 256x256 game-ready sprites.

  Step 4: Scale production

  Once the workflow is validated with 2-3 characters:
  - Batch all 35 hero refs through Tripo3D (need paid plan or do in batches over months)
  - Batch Mixamo rigging (manual but ~5 min per character)
  - Batch render + post-process (automated, ~30s per character)

  Step 5: UE5 integration

  Import final sprites into Paper2D flipbooks for in-game use.

## Issues / Blockers
- **ComfyUI abandoned**: AI image-by-image generation (~4700 sprites) had fatal frame-to-frame consistency issues. 3D pipeline (Tripo3D+Mixamo+Blender) is the validated approach.
- **Non-humanoid monster rigging**: Mixamo only auto-rigs humanoids. Need a different rigging approach for monsters like Poring, Baphomet, etc. (manual Blender rigging? separate pipeline?)
- **Equipment layer system not started**: Hair, weapon, headgear, shield, garment need separate atlas layers that composite at runtime. Architecture designed but not built yet.
- **Animation standardization**: 20 FBX anims validated for Swordsman but need to define which subset each class gets (e.g., mages don't need 2H attack, archers don't need block)
