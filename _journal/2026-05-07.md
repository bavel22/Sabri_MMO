### `respawnMs` per-template ranges (ro_monster_templates.js + Enemy_System.md)

**Format extension**: `respawnMs` now accepts `[min, max]` for uniform random delay, in addition to the existing fixed-number form. Resolved by `resolveRespawnMs()` in index.js:

```js
function resolveRespawnMs(value) {
    if (Array.isArray(value) && value.length === 2) {
        return value[0] + Math.floor(Math.random() * (value[1] - value[0] + 1));
    }
    return value || 15000;
}
```

## Key Decisions

- **`resolveRespawnMs` lives at index.js ~line 100** â€” single helper, used at exactly two timer-arming sites (status-event L2793 + combat-death L28799). Three other touch-points just copy the value; don't wrap them. Locked into a memory entry to prevent future regressions.
- **Range form `[min, max]` is for non-MVP rare named / farming spawns ONLY** â€” MVPs and bosses keep their hardcoded variance in the death handler. Don't generalize.
- **Default to `[300000, 600000]` (5-10 min) for plants** â€” long enough that a farmer can't camp the spot, short enough that a wandering player still encounters the species in a reasonable session.
- **`spawnPool[]` migration is per-zone, not flag-day** â€” Prontera South switched today, but `prt_dungeon_01` and `prontera` (town) keep `enemySpawns[]` because their density is hand-tuned. Mixed coexistence supported by design.
- **Repurpose existing portals over adding new ones** â€” the `prtsouth_to_dungeon` portal at `(-650, 9560, -20)` was already in the level; just point it elsewhere. Players keep using the same physical portal location, no level edits needed.
- **Two-layer movement validation** â€” fast-path AABB for zone-data deep, fallback navmesh validation for `AWaterArea` mixed-mode. Both run; the navmesh check is the backstop catching everything the AABB list missed.
- **Snap-and-warn over reject** â€” designer typos (spawn point inside deep water) are common; snapping with a logged warning is friendlier than silently dropping the spawn.
- **Filed `feedback-respawn-range-syntax.md`** so the resolver/copy-site distinction survives future contributions. The 5 touch-points are subtle and easy to miss.
