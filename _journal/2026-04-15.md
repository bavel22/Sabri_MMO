## Key Decisions

- **Share one atlas across recolor variants via runtime tint multiply** — rather than re-rendering + re-packing an entire atlas per color. Saves disk, memory, artist time, and keeps RO's dozens of tinted variants (Andre/Deniro/Piere/Vitata, Wolf variants, Chonchon variants) cheap to add. The body material already has a `TintColor` path from the hair system, so we reuse it
- **Spawn billboard sprites, not decals, for ranged enemy attacks** — billboards always face camera, decals project onto surfaces and read poorly from horizon-level angles. Reusing `ASpriteCharacterActor` with the attacker's own sprite class keeps the visual consistent with the creature's style
- **Add enemies to spawn list BEFORE atlases are rendered** — flushes out missing-atlas handling early and lets us prioritize the rendering queue by what's actually spawning. Useful forcing function for the client fallback path
- **Surgical doc edits, not rewrites** — plan docs that represent completed work get a "STATUS: COMPLETE" header; original multi-phase breakdown preserved as historical reference
- **Dashboard is the single source of truth** for "what's done" — plans should cross-reference the dashboard rather than duplicate the list
