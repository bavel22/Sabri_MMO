## What Got Done

### Enemy Attack Lunge System (EnemySubsystem.cpp, +98 lines)
- **Melee lunge**: wind-up (pull back 20 units) → lunge toward target (60 units max, clamped to 40-unit clearance) → snap back. Uses `ServerTargetPos` + timer chain (0.08s wind-up, 0.10s hold, 0.15s return). Only triggers within `MeleeThreshold` (250 units)
- **Ranged ground attack**: for enemies attacking beyond melee range (Mandragora vine whip), spawns a temporary root tendril / cracked earth decal at target's feet with 0.8s lifetime + random rotation
- Safe: `TWeakObjectPtr` guards, death/despawn checks, per-enemy timers

### Ground Strike VFX (SkillVFXSubsystem, +24 lines)
- New `SpawnGroundStrikeEffect(FVector Location)` for earth-colored upward burst at target feet
- Uses `NS_Dark_Stone_Impact` Niagara system with green tint `(0.3, 0.55, 0.15)` for vine attacks
- Falls back to `NS_AutoAttackHit` if dark stone VFX missing

### Render Pipeline Improvements (render_monster.py, +64 lines)
- `--thicken` CLI arg: adds Solidify modifier for thin geometry (wings/fins) that would be invisible edge-on. Centered offset, even thickness on angled faces
- `interpolate_lunge()`: linearly interpolates lunge offset at any frame from sparse keyframe dict
- `lunge_offsets` support in `render_all()`: physically translates mesh toward camera during attack frames, creating a visible ram/jolt effect correct from all 8 directions

## Key Decisions

- **Melee lunge uses ServerTargetPos, not actor position** — sprite interpolation handles the smooth movement, and the snap-back is clean
- **Ranged enemies get decal-based attacks** instead of lunge — Mandragora vine whip spawns a ground effect at target feet rather than jolting the sprite forward
- **Thicken via Solidify modifier** rather than geometry duplication — simpler, reversible, works on all mesh types
