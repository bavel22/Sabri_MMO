## What Got Done

### Sprite Rendering — Full Sprite-vs-World System (major, FINAL)
Built and iterated through MANY approaches to get sprite-vs-world rendering correct. Final solution combines:

**Sprite material** (`SpriteCharacterActor.cpp::CreateSpriteMaterial`):
- `BLEND_Translucent` + `bDisableDepthTest = 1` + `AllowTranslucentCustomDepthWrites = 1`
- Opacity = binary alpha (always writes stencil regardless of occlusion)
- Emissive = `lerp(SpriteColor, SceneColor, FinalOccluded)` where `FinalOccluded` combines two trace results + per-pixel depth check
- Two scalar parameters: `FeetOccluded`, `HeadOccluded` (set per frame from C++)
- `UMaterialExpressionSceneColor` for sampling the wall behind the sprite
- `UMaterialExpressionPixelDepth`/`SceneDepth` for per-pixel "behind wall" check (50-unit threshold)
- `bEnableResponsiveAA = 1` to reduce TAA smoothing on hard edges
- Layer sorting via `TranslucencySortDistanceOffset` (replaces old X-offset SetRelativeLocation)

**Per-frame line traces** (`SpriteCharacterActor.cpp::UpdateOwnerTracking`):
- Camera → actor.feet (Loc + 8 up): sets `FeetOccluded`
- Camera → actor.head (Loc + SpriteSize.Y up): sets `HeadOccluded`
- Both use `ECC_Visibility` channel, ignore self + owner
- Updated on each layer's MID (9 layers)

**Post-process cutout** (`PostProcessSubsystem.cpp::CreateCutoutMaterial`):
- `MD_PostProcess` with `BL_SceneColorAfterTonemapping`
- Reads `PPI_PostProcessInput0` (scene color) + `PPI_CustomStencil`
- Where stencil ≠ 1 → fade scene color toward 35% brightness
- Where stencil == 1 → leave alone (player area always full brightness)
- Re-applied after `ApplyZonePreset` clears blendables
- `r.CustomDepth = 3` set at startup

**Behavior matrix:**
- Sprite in front of any geometry → full sprite color (FeetOccluded=0)
- Sprite fully behind a wall → silhouette via post-process (HeadOccluded=1, sprite emits scene color)
- Sprite partially behind a low wall (head sticking out) → per-pixel depth check; lower body silhouette, head/upper body sprite art
- Sprite quad clipping into nearby wall → no clipping (bDisableDepthTest=1 + 50-unit Behind threshold ignores small depth differences)

**Other fixes** (`SpriteCharacterActor.cpp`, `SpriteAtlasData.h`):
- `SpriteDepthOffset` = 0 (was 125) — quad vertices at local X=0
- Hardcoded 96-unit capsule half-height for player sprite Z positioning (matches local player capsule, works for remote BP regardless of its actual capsule size)
- `OtherCharacterMovementComponent` uses hardcoded 96 to match
- Motion blur disabled in `PostProcessSubsystem::ApplyZonePreset` (`MotionBlurAmount = 0`) — translucent materials don't write velocity, so motion blur using stale world velocities was smearing sprites
- Local player name tag registered on sprite actor (was pawn) via `LocalSprite` weak ptr on `ASabriMMOCharacter`
- Health bar uses `LocalSprite` for local player position (sprite pos for remote)
- `GetSpriteScreenBounds`: `GSpriteCameraDepthOffset = 0` — projects from actor position directly, scales correctly with zoom
- `CameraSubsystem` line-trace-and-hide-actor occlusion REMOVED entirely (replaced by post-process cutout)

## Key Decisions

- **bDisableDepthTest = 1 + line traces + per-pixel depth**: After trying many approaches (PDO, SDPG_Foreground, WPO, surface normal check, SceneCapture, etc.), the line-trace + per-pixel depth combo is the only one that handles all cases correctly without breaking any. Surface-normal check alone was inconsistent (vertical walls let sprite through, sloped surfaces blocked it).
- **Two traces (feet AND head)**: Single feet trace caused full silhouette even when the head was sticking out above a low wall. Adding head trace gives partial-occlusion handling where the per-pixel depth check kicks in.
- **Emissive = lerp(SpriteColor, SceneColor) instead of opacity = 0**: Setting opacity to 0 made the sprite fully invisible (custom depth pass also clipped, no stencil). Emitting scene color makes the sprite "vanish" visually while keeping the stencil written.
- **Fixed 96 capsule half-height (not dynamic)**: All player positions are broadcast from the local player's capsule center. Using the remote BP's actual capsule size caused a mismatch (typically 88 vs 96 = 8 units of floating).
- **Audio research before any audio code**: Resisted the temptation to start grabbing wav files first. The legal-status finding ($4M Gravity v. NovaRO judgment) materially changes the architecture: build the tables/lookup conventions exactly like RO, but plan for original wavs from day one. Cheaper to learn this before writing code than after.
- **Don't touch the misspellings**: every monster file in RO has at least one Korean → English transcription artifact. Preserving the exact original names means file references work without a "fix" map and matches every existing community resource.
