Two strategies survive: "rotated 30°" + "3quarter with side lighting."
- `hunyuan_texture_test.py` — confirms 5K-face decimate-then-texture is 30× faster than texture-then-decimate, with no visible difference at 15° MMO camera FOV.

### Master pipeline `_tools/hunyuan_asset_pipeline.py` (23:04)

48,186 bytes. The canonical asset generation function — 6 stages per asset (~4-5 min on RTX 5090):

1. **Generate input images** — multi-input mandatory (2 perspective strategies × 2 SDXL seeds = 4 candidates). Each strategy is a long prompt with explicit margin/single-object/no-shadow/no-frame negatives.
2. **Mesh-gen seed sweep** — `Hy3DMeshGenerator` + `Hy3D21VAEDecode` (mc_algo="mc") on each input. ~35s per candidate.
3. **Pick best** — quality-aware scoring: `score = depth_ratio × volume`, with `cube_artifact (vol > 5.0) = 0.001` so any non-cube alternative wins.
4. **Decimate first** — `Hy3D21MeshlibDecimate target_face_num=5000` (~5s). Critical: decimate-then-texture is 30× faster.
5. **Texture decimated mesh** — `Hy3DMeshUVWrap` + 6-camera multi-view bake (front/back/top/bottom weighted higher than sides) + `InPaint`. ~40s.
6. **Save** — `06_final.glb` (5K-face textured, ~1.5 MB) + `manifest.json` (full pipeline log: scores, attempts, reroll history).

Ancillary helpers in the same file: `validate_masked_input` (3 checks: 3-92% coverage, < 2 cut-off edges, single connected blob > 80% of mass), `is_good` (gates ≥ 0.15 score, vol ≤ 5.0, vol ≥ 0.05, depth ≥ 0.4), `VARIATION_SUFFIXES` (8 cycle-through prompt suffixes for reroll attempts).

`hunyuan_cleanup.py` (14,994 bytes) — pulls all assets with score < 0.15, rerolls each up to 8 times with cycling `VARIATION_SUFFIXES`, backs up the previous best to `_history/<asset_id>/attempt_N/` *outside* `asset_dir/` (rmtree-safe — restores the better version if the reroll regresses).

## Key Decisions

- **Multi-input is mandatory** — single-input failed 60-70% on first attempt. Two strategies × two seeds = 4 candidates × 70% first-attempt success. Worth the extra ~2 min per asset.
- **Decimate-FIRST then texture** — 30× speedup. Visual difference at isometric 15° camera is undetectable.
- **Quality-aware scoring with `cube_artifact = 0.001`** — preserves best version across rerolls. Without the artifact penalty, cube-shape failures (volume 5.5+) beat legitimate assets (volume 0.3-1.0).
- **Backup OUTSIDE `asset_dir`** — `rmtree-safe`. Rerolls that crash mid-write don't lose the previous best.
- **`mc_algo="mc"` is the only working mesh algorithm** on torch 2.12 / sm_120 / RTX 5090. `dmc` returns None.
- **`PYTHONIOENCODING=utf-8` + `PYTHONUTF8=1`** must be set in every launcher. Without them, ComfyUI crashes on cp1252 output from custom nodes.
- **Always run ComfyUI under `comfyui_supervisor.py`** — auto-restart pattern survives all observed crash modes (exit 5, OOM, KJNodes API drift).
- **Pin KJNodes to commit `37a0973`** — newer KJNodes broke `comfy_api.latest` during the morning runs.
- **Skip variants for bad bases** — if `is_good(asset) == False`, don't waste GPU minutes generating colored siblings of garbage.
- **Inherent failures: stop after 8 attempts** — the AI fundamentally can't make thin pole-like or sparse-foliage shapes volumetric. After 8 rerolls without improvement, mark as "use procedural alternatives" and move on.
- **Idempotent skip-if-done everywhere** — Windows reboot mid-batch is non-fatal. Every orchestrator + cleanup script + variant generator skips assets where `06_final.glb` exists with the right pipeline metadata.
