## What Got Done

### Session 1: Login Screen Background
**Goal**: Add a visual login background and fix resolution/sizing in standalone mode.
**Accomplished**: Full login background system with deferred loading.

- Deferred texture loading via `TryLoadBackgroundTexture()` retry pattern (0.2s retries until texture > 64px)
- `FSlateBrush` with `SetResourceObject()` (NOT deprecated `FSlateDynamicImageBrush`)
- `SScaleBox` with `ScaleToFill` + HAlign/VAlign Center — fills viewport, preserves ratio, crops edges
- Texture settings: UserInterface2D, Power of Two = None, Never Stream = true
- Camera `bConstrainAspectRatio=true`, `AspectRatio=16/9` on FollowCamera for 3D scene protection

**Key learning**: In standalone mode, viewport reports 0x0 and `StaticLoadObject` returns 32x32 placeholder during `OnWorldBeginPlay`. Must defer widget creation with retry.

### Session 2: ESC Menu & Character Select Return
**Goal**: Add RO Classic ESC menu with character select return flow.
**Accomplished**: Full ESC menu system with return-to-character-select.

- `EscapeMenuSubsystem` + `SEscapeMenuWidget` — RO Classic "Select Option" popup
- Dead/alive button states (respawn visible only when dead)
- `player:leave` two-phase handler: sync cleanup (delete from maps, broadcast) BEFORE await, then async DB saves
- `bReturningToCharSelect` flag on GameInstance — LoginFlowSubsystem skips to character select
- Login background at Z=200 covers all game widgets (Z 5-50) instead of removing them
- Static `FButtonStyle` required for SButton (stores pointer, not copy)

**Key learning**: `RemoveAllViewportWidgets()` does NOT clear viewport content widgets. High Z-order background overlay is the reliable solution.

### Session 3: Right-Click Player Context Menu
**Goal**: Add right-click context menu on other players (Trade, Party Invite, Whisper).
**Accomplished**: Full system working, all 3 actions route to existing subsystems.

- Camera drag vs quick-click detection via `CameraSubsystem::bDidRotateThisClick` flag
- `PlayerInputSubsystem::OnRightClickFromCharacter()` → cursor trace → `OtherPlayerSubsystem::GetPlayerIdFromActor()` → `ShowPlayerContextMenu()`
- `FSlateApplication::PushMenu()` for `FMenuBuilder` popup
- Trade → `TradeSubsystem::RequestTrade(charId)`
- Party Invite → `PartySubsystem::InvitePlayer(name)`
- Whisper → `ChatSubsystem::StartWhisperTo(name)` → prefills `/w "Name" ` in chat input
- Guard conditions: dead, hidden, non-player actors, self excluded

**Key learning**: No server changes needed — all 3 actions already had server handlers from Storage/Trading sessions.

### Session 4: Item Icon Mapping
**Goal**: Map all 6169 items to their generated icon assets in UE5.
**Accomplished**: Updated `InventorySubsystem::GetOrCreateItemIconBrush()` with subfolder-based icon loading.

- Changed `IconAssetMap` from flat names to `subfolder/AssetName` format
- Path resolution: `/Game/SabriMMO/Assets/Item_Icons/{subfolder}/Icon_{AssetName}`
- Added categories: consumables, etc_crafting, weapons, armor, cards, ammo, etc.
- Underscore fallback: `icon_name` → tries `Icon_{icon_name}` directly if not in map
- All 6169 items now have icon paths

## Key Decisions
- **Deferred widget creation** — Standalone mode timing issues mean all login widgets must use retry pattern, not direct creation in OnWorldBeginPlay
- **Z-order overlay instead of widget removal** — Z=200 background covers game UI during login/char select instead of unreliable RemoveAllViewportWidgets()
- **Mouse delta flag for right-click** — `bDidRotateThisClick` is simpler and more reliable than time-threshold for distinguishing camera rotate from context click
- **Subfolder icon organization** — Icons organized by category subfolder in UE5 Content, mapped via `IconAssetMap` in C++
