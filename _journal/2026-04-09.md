## What Got Done

### Ground Item System ‚Äî Server (index.js, ~207 new lines in globals + helper functions)
- **Core data structures**: `groundItems` Map (groundItemId ‚Üí GroundItem), `zoneGroundItems` Map (zone ‚Üí Set of IDs)
- **Ownership phases** (RO Classic pre-renewal):
  - Normal drops: 0-3s #1 damage dealer only, 3-5s top 2, 5-7s top 3, 7s+ free-for-all
  - MVP drops: 0-10s MVP winner only, 10-20s top 2, 20-22s top 3, 22s+ free-for-all
- **Scatter system**: SE/W/N offset cycle (matches rAthena mob.cpp), 50 UE unit scatter distance
- **Party item share**: `party:change_item_share` and `party:change_item_distribute` socket handlers
- **Despawn timer**: 60s lifetime, 5s check interval, batch despawn broadcast per zone
- **Pickup validation**: Range check (250 UE units), dead/sitting/vending/trading/overweight guards
- **Socket events**: `item:spawned_batch`, `item:despawned_batch`, `item:ground_list`, `item:pickup`, `item:pickup_error`, `item:picked_up`
- **Drop integration**: `spawnMonsterGroundDrops()` wired into enemy death handler with damage ranking from `inCombatWith`

### Research Document
- Created `docsNew/05_Development/Ground_Item_And_Drop_System_Research.md` ‚Äî RO Classic drop mechanics, ownership tiers, party loot modes, pickup range, despawn timing

## Key Decisions

- **250 UE unit pickup range** ‚Äî generous to account for 30Hz position broadcast delay (client position can lag ~33ms behind server)
- **Scatter offset cycle SE/W/N** ‚Äî matches rAthena mob.cpp scatter pattern for multi-drop consistency
- **Ownership uses damage ranking, not last-hit** ‚Äî matches RO Classic behavior where top damage dealers get priority
- **Party share mode checked at pickup time** ‚Äî party can change item share mode between drop and pickup
