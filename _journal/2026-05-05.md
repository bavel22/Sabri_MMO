### `_tools/hunyuan_v3_regional.py` (00:28)

140,537 bytes — the largest pipeline file yet. Major additions over `hunyuan_asset_pipeline.py`:

- **Per-region asset lists** — separate dicts for `MOSCOVIA_ASSETS`, `UMBALA_ASSETS`, `ECLAGE_ASSETS`, etc. Each entry is `{ id, category, subject, regional_palette }` where `regional_palette` modifies the SDXL prompt with region-specific color words (e.g. `"snow-dusted weathered grey wood with rusted iron bands"` for Moscovia, `"warm green-painted bamboo with woven palm-frond accents"` for Brasilis)
- **Regional negative prompt extensions** — Moscovia gets "no Asian architecture," Brasilis gets "no temperate trees, no winter," etc., to keep SDXL from drifting between regions
- **Lower per-asset reroll budget (5 vs 8)** — regional assets are usually pickier prompts, and 5 attempts is enough to find a working one or skip and move on
- **Regional grouping in output dir** — `generated_assets/regional/<region>/<category>/<asset_id>/` so the regional library doesn't dilute the broad library

The file is ~3× the size of the master pipeline because it embeds the asset definitions inline rather than loading them from JSON. Trade-off: easier to read in one place, harder to swap region lists. May refactor to JSON loading later.

## Key Decisions

- **Document regions in `_meta/`, not `docsNew/05_Development/`** — the regional asset list is a designer reference, not a system spec. `_meta/` is where world-building references live (alongside the existing landscape / materials / scatter / decals / lighting guides).
- **Per-region prompts with regional palette overrides, not hand-tuned per-asset SDXL prompts** — the master pipeline already validates SDXL output. Regional variation comes from a few extra color/style words on top of the base prompt template, not from rewriting the prompt structure.
- **Lower reroll budget for regional (5 vs 8)** — niche prompts that don't first-roll are usually fundamental SDXL limitations, not seeds. Don't burn 8 attempts when 5 will tell you whether the model can produce it.
- **Keep regional output separate from the broad library** — `generated_assets/regional/<region>/` rather than mixed in with vegetation/architecture/props. Future zone builders pull from broad + their region's bucket.
