## What Got Done

### Session 1: Kafra Storage
**Goal**: Build account-shared 300-slot Kafra Storage system.
**Accomplished**: Full system implemented and working.

**Server (10 socket handlers)**:
- 6 storage handlers: `storage:open/close/deposit/withdraw/cart_deposit/cart_withdraw`
- 4 inventory QoL handlers: `inventory:split/sort/auto_stack`, `storage:sort`
- `sendStorageContents()` helper, atomic transactions with FOR UPDATE locks
- Rate-limited (5/sec split, 2/sec sort/auto-stack)

**Client (StorageSubsystem + SStorageWidget)**:
- StorageSubsystem (Z=21) with 10-column grid, tabs, toolbar
- Drag-drop deposit/withdraw between inventory and storage
- Alt+RightClick quick deposit/withdraw
- Shift+Click split/quantity popup
- Search filter (case-insensitive name filter)
- Sort buttons (Type/Name/Weight)
- Auto-Stack button (merges stackable duplicates)
- Escape key closes both Storage and Inventory windows

**Database**:
- `account_storage` table keyed on `user_id` (account-shared, not per-character)
- `compounded_cards` column as JSONB (not TEXT)
- Migration: `database/migrations/add_account_storage.sql`

**Integration**:
- SKafraWidget "Use Storage" button (40z fee)
- InventorySubsystem CompleteDrop for StorageSlot
- CharacterData.h storage enums

**Bugs found and fixed**:
1. `player.userId` not stored — JWT `decoded.user_id` was verified but discarded. Fixed by saving to `socket._authenticatedUserId` → player object
2. `compounded_cards` TEXT vs JSONB mismatch — character_inventory uses JSONB but account_storage was TEXT. Fixed to JSONB + JSON.stringify()
3. Button used `GEngine->GetCurrentPlayWorld()` — unreliable in Slate context. Fixed to `KSub->GetWorld()`
4. GridTop offset bug — adding toolbar shifted grid down but `GetSlotIndexFromPosition()` hardcoded offsets weren't updated, causing all clicks to hit wrong slot. Fixed: Inventory GridTop 24→46, Storage GridTop 50→71

### Session 2: Player-to-Player Trading
**Goal**: Build RO Classic direct trading system with anti-scam protection.
**Accomplished**: Full system implemented and working.

**Server (9 socket handlers)**:
- TradeSession class (~line 535) with state machine (pending → trading → locked → finalizing)
- 9 handlers: `trade:request/accept/decline/add_item/remove_item/set_zeny/ok/trade/cancel`
- `cancelTrade()` helper with 13 call sites (death, disconnect, zone change, etc.)
- Atomic PostgreSQL transaction with FOR UPDATE locks — delete-before-insert transfer
- All trade queries use `client.query()` (transaction connection), NOT `pool.query()`
- `trade_logs` table (JSONB items, zeny, zone) for audit trail

**Client (TradeSubsystem + STradeWidget)**:
- TradeSubsystem with 11 event handlers
- STradeWidget (Z=22) — dual-panel layout (My Items / Partner Items), 10 slots each
- STradeRequestPopup (Z=30) — inline accept/decline popup
- Two-step confirmation: OK (lock items) → Trade (finalize) — anti-scam
- Lock resets on ANY modification after locking (prevents bait-and-switch)
- SelfHitTestInvisible wrapper so inventory remains clickable during trade
- Movement locked during trade (DisableMovement pattern from VendingSubsystem)
- Right-click partner items → ItemInspectSubsystem::ShowInspect()

**Chat integration**:
- `/trade PlayerName` command in ChatSubsystem

**Database**:
- `trade_logs` table, `items.trade_flag` column (auto-created at startup)
- Migration: `database/migrations/add_trade_system.sql`

**Bugs found and fixed**:
1. `trade_flag` column missing on items table — added auto-create at server startup
2. Trade widget blocked all viewport clicks — switched to SelfHitTestInvisible wrapper
3. Item slots had no icons — added SImage with TAttribute lambda polling InventorySubsystem brush cache
4. Partner items not inspectable — added right-click → FTradeItem→FInventoryItem converter

## Key Decisions
- **Account-shared storage** — keyed on `user_id` not `character_id`, matches RO Classic
- **JSONB for compounded_cards everywhere** — learned from bug: TEXT/JSONB mismatch causes insert failures
- **Two-step trade confirmation** — OK locks items, Trade finalizes. Any modification resets locks
- **Movement locked during trade** — reuses VendingSubsystem's DisableMovement pattern
- **SelfHitTestInvisible** — trade widget wrapper so inventory drag-drop still works underneath
- **QoL batch** — added split/sort/auto-stack/search to both inventory and storage in same session

## Issues / Blockers
- **Right-click context menu on other players** — Trading currently requires `/trade PlayerName` chat command. Right-click on players in world needs PlayerInputSubsystem right-click routing (deferred)
- **GridTop offset fragility** — When adding new rows to Slate widget layouts, all hardcoded pixel offsets in `GetSlotIndexFromPosition()` must be manually updated. Consider computing offsets dynamically in the future
- **Character floating off ground** — character seems to be floating a bit, can clip through terrain/objects when moving north
- **No ESC → character select** — Escape should return to character select screen (not implemented)
