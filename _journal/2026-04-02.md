**Goal**: Build the complete RO Classic map system from scratch.
**Accomplished**: All 9 phases completed.

**Minimap** (Phases 2, 4, 9):
- MinimapSubsystem + SMinimapWidget — 134x134px compact widget (128 map + 3px gold frame)
- SceneCapture2D at 16 FPS, orthographic, 256x256 render target, 5000 units above player
- 5 zoom levels (1.0x to 8.0x), 3 opacity states (Tab cycles)
- Draggable via left-click drag with DPI-correct offset
- Entity dots: white player (6px), orange enemies (3px), white other players (3px), pink party (5px), blue NPCs (4px), red warps (5px), guide marks (blinking crosses)
- Double-frame border: gold outer + dark inner, zone name overlay inside map

**World Map** (Phases 3, 8):
- SWorldMapWidget — fullscreen grid overlay
- 12x8 grid from `ro_world_map_data.js`, 62 unique zones, category tinting (gold=town, green=field, red=dungeon)
- Hover tooltips, zone name toggle (N), monster info toggle (Tab), party cross-zone dots
- Keyboard: M=toggle, Esc=close, Tab=monsters, N=zone names

**Loading Screen** (Phase 5):

## Key Decisions
- **Minimap uses SceneCapture2D** — real overhead camera, not a pre-made texture. 16 FPS is enough.
- **World map uses procedural grid** — colored rectangles until a proper hand-painted map texture is generated
- **Loading screen is inline class** — SLoadingScreenOverlay lives in ZoneTransitionSubsystem.cpp, not a separate file
- **Guide NPC is server-driven** — facility locations and names come from `ro_zone_data.js`, not hardcoded on client
- **Preferences on GameInstance** — zoom/opacity survive zone transitions without server round-trip
