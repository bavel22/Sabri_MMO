## What Got Done

### `AWaterArea` mixed-mode rewrite (07:07 → 07:08)

**Single actor, auto-detected deep cells, navmesh-correct.** Replaces the old "either shallow or deep" gate with a per-cell deep map computed at `OnConstruction` (and re-run at `BeginPlay` to handle runtime-spawned actors and editor sessions where terrain changed).

New `UPROPERTY` fields (`WaterArea.h`):

| Field | Type | Default | Use |
|---|---|---|---|
| `bAutoDetectDeep` | `bool` | `true` | When true, raycasts auto-determine deep cells. False falls back to whole-area `bIsDeep`. |
| `bIsDeep` | `bool` | `false` | Manual override / fallback when auto-detect is off |
| `DeepDepthThreshold` | `float` | `200.f` | UE units below water surface to mark a sample as deep (~50 UE = 1 RO cell, so 4-cell depth) |
| `DepthSampleResolution` | `int32` | `16` | NxN grid of downward raycasts. 16 → 256 samples |
| `ShallowColor` | `FLinearColor` | `(0.08, 0.18, 0.5, 0.3)` | Tier 1 color (clear blue) |
| `DeepColor` | `FLinearColor` | `(0.03, 0.08, 0.35, 0.5)` | Tier 2 color at `DepthFalloff` (~250 UU) |
| `AbyssColor` | `FLinearColor` | `(0.005, 0.015, 0.05, 1.0)` | Tier 3 — extreme depth, near-black navy |
| `AbyssOpacity` | `float` | `1.0` | Fully opaque at abyss depth (no see-through) |
| `AbyssDepth` | `float` | `900.f` | Depth where water reaches abyss tier |
| `WaveSpeed` | `float` | `0.5f` | Wave animation speed |

Three-stop depth gradient inside the runtime material:
```
Stage 1: lerp(ShallowColor, DeepColor,
              saturate(WaterDepth / DepthFalloff))      // 0 → 250 UU
Stage 2: lerp(Stage1Color, AbyssColor,
              saturate((WaterDepth - DepthFalloff) /
                       (AbyssDepth - DepthFalloff)))    // 250 → 900 UU
```

Color, opacity, roughness, specular, and ripple amplitude all respond to depth. Abyss reaches `AbyssOpacity` = 1.0 — fully opaque, hides everything below.

### Raycast deep detection + greedy rectangle merge

`PerformDepthScan()` — fires `DepthSampleResolution²` (default 256) downward `LineTraceSingleByChannel` calls from a grid above the water actor. Each cell with `WaterLevel - HitZ ≥ DeepDepthThreshold` is flagged deep. No-floor cells (raycast misses) default to deep.

`SpawnDeepNavBoxes()` — coalesces deep cells into the smallest set of axis-aligned rectangles using a greedy merge:
1. Walk the grid row-major, find an unmerged deep cell
2. Greedily extend right (same row) until a non-deep cell or grid edge
3. Greedily extend down (column-by-column) until any row's matching range is non-deep
4. Mark all merged cells, spawn one `UBoxComponent` per rect

Each `UBoxComponent`:
- `bCanEverAffectNavigation = true`
- `AreaClassOverride = UNavArea_Null::StaticClass()`
- Marks the navmesh under the rect as un-navigable

Multiple overlapping cuts stack cleanly — the navmesh just sees them as one big un-navigable region.

`player.activeWaterAreas` is a `Map<areaId, { isDeep }>`. `inWater = (Map.size > 0)`. Overlapping `AWaterArea` actors don't break state — leaving one while still inside another keeps `inWater=true`. The two-layer movement validation (server-side) lands tomorrow:

2. **Off-navmesh reject** (general): general navmesh validation, snap distance > 100 UU. Catches per-cell deep regions auto-detected by `AWaterArea` mixed-mode without per-cell server data.

## Key Decisions

- **Mixed-mode in one actor** — original research suggested separate shallow / deep volumes. Collapsed into a single `AWaterArea` because (a) a canal is conceptually one body of water and (b) the auto-detect makes deep tracking automatic from level geometry rather than designer-configured per-cell.
- **DefaultLit translucent over Single Layer Water (SLW)** — SLW was considered but the runtime material build path was simpler with translucent + manual `SceneDepth - PixelDepth` for the depth gradient. Provides equivalent depth-aware coloring/opacity at lower complexity. Custom HLSL was tested and confirmed *NOT* to animate at runtime (HLSL is compiled once and cached) — only standard `UMaterialExpression*` nodes update per-frame. Locked into standard nodes.
- **Three-stop gradient** — shallow → deep → abyss. Two stops weren't enough: shallow → deep alone gave a uniform "rich blue" that obscured the floor. Adding the abyss tier creates a clear "do not enter" visual cue at the deepest cells, which matches the navmesh cut.
- **Auto-detect at `OnConstruction` AND `BeginPlay`** — `OnConstruction` covers editor placement, `BeginPlay` covers runtime-spawned actors and editor sessions where terrain edits happened after the actor was placed.
- **Greedy rect merge over per-cell boxes** — 256 boxes is workable but inefficient. Greedy merge typically reduces 200 deep cells → 8-15 rectangles. UE5 navmesh handles many small cuts fine but fewer is faster + cheaper to debug.
- **`bCanEverAffectNavigation = true` is mandatory** on each nav box, even though the parent actor's TriggerComp has it false. The child boxes are the only thing that should cut the navmesh; the trigger is for overlap events only.
- **Spawn snap-and-warn over spawn-reject** — when a spawn point lands inside deep water, snapping to nearest navmesh + logging a warning is friendlier than dropping the spawn. Catches designer typos automatically.
