### Client Endpoint Config Design — `Client_Endpoint_Config_Design.md` (27 KB)
- Problem statement verified from source (`MMOGameInstance.h:118`, `MMOHttpManager.cpp:25-32`, `MMOGameInstance.cpp:104-111`, `server/src/index.js:34142`)
- **Resolution order**: `Final URL = CLI ?? SaveGame ?? Compile-time default`
  - CLI: `FParse::Value(FCommandLine::Get(), TEXT("server="), ServerArg)` → `SabriMMO.exe -server=staging.sabrimmo.com:443`
  - SaveGame: new `UOptionsSaveGame::CustomServerUrl` UPROPERTY, hidden advanced panel
  - Compile-time: `SABRI_DEFAULT_MASTER_URL` preprocessor define, set in `SabriMMO.Target.cs` via `GlobalDefinitions.Add("SABRI_DEFAULT_MASTER_URL=\"https://sabrimmo.com\"")`
- TLS/WSS migration plan: UE5.7 HTTP module uses system CA store by default (no config needed); SocketIOClient plugin needs to be rebuilt with WSS support enabled
- Version handshake:
  - `/health` returns `{ status, version, timestamp, message }` (new `version` field)
  - Client fetches `/health` before socket connect; compares against bundled `SABRI_CLIENT_VERSION`
  - Mismatch → UX: "Server is on v0.5.0, your client is v0.4.3. [Download update]" + link to latest itch.io release
- **C++ diffs included inline**: `MMOGameInstance.h` (+2 UPROPERTY), `MMOGameInstance.cpp::Init` (resolution order), `MMOGameInstance.cpp::GetServerSocketUrl` (TLS scheme swap), `MMOHttpManager.cpp` (fallback chain), `UOptionsSaveGame` (new field)

## Key Decisions

- **Self-host on own hardware through Scenarios A and B** — $250 N100 mini PC or repurposed spare PC beats any VPS under 20-50 CCU. Only move to Hetzner CCX23 US / AX41 dedicated at 100+ CCU or if home upload saturates. Saves $20-40/mo × 6-12 months before needing to move at all
- **Tailscale mesh for 5 friends, Cloudflare Tunnel for 10-50 invitees** — both sidestep CGNAT and port forwarding. Tailscale is invite-only by design (perfect for Scenario A) but capped at 6 users free. Cloudflare Tunnel unlimited, auto-TLS, CGNAT-friendly, ~20-60ms latency overhead
- **Reject Steam for distribution** — Nov-2024 private branch feature still requires public "Coming Soon" page during review (2-week minimum), which exposes name + capsule to Gravity Co.'s IP monitoring. Dreadmyst 2026 NCsoft DMCA precedent. itch.io Restricted with bulk download keys is the invite-only answer
- **Reject AWS/Azure/GCP for game server** — egress cliff ($441/mo on AWS at 5 TB vs $0 on Hetzner/OVH) and burstable-tier CPU throttling will wreck the 50ms tick under sustained load. Wrong tool for MMO backend
- **Reject Upstash PAYG Redis** — 20 Hz tick × 100 CCU × $0.20/100K commands = **~$13,000/month**. Fixed tier ($10/mo, unmetered commands) is fine. This is the single biggest accidental-bankruptcy risk on any managed-stack build
- **Endpoint config: SaveGame + CLI + compile-time default, NOT GConfig INI** — honors CLAUDE.md "Use SaveGame for custom persistence, NOT GConfig custom ini" rule (GConfig doesn't load reliably in standalone). Compile-time macro bakes production URL into binaries; CLI is for QA overrides; SaveGame is for tech-savvy users debugging
- **Don't implement yet** — scope today was research + decision. Actual code changes to `MMOGameInstance` and `/health` get their own session with skill-gated context loading
- **Version handshake via `/health`, NOT socket handshake** — simpler (REST), can be verified pre-socket-connect, keeps socket protocol unchanged, makes "please update" UX trivial (block login screen until version matches)
- **Skill-selection rule formalized in CLAUDE.md** — "prefer loading MORE skills than strictly necessary" with reasoning. This was previously implicit / inconsistent; codifying prevents the "forgot to load `/sabrimmo-persistent-socket` on a zone-transition bug" class of mistake
